# Misheard Music — бриф для разработки

Party-игра «ослышки»: команда угадывает настоящую строчку/название песни по
фонетически искажённой версии («как слышится»). Механика по типу «Элиас» —
pass-and-play на одном телефоне.

**Формат: мобильный веб (сайт).** Прототип `reference-prototype.html` —
рабочий референс всей механики и дизайна. Открой его в браузере и прогони
полный цикл игры прежде чем писать код.

---

## Игровая механика (зафиксирована, не менять без обсуждения)

- 2–8 команд, у каждой название (редактируется на экране настройки, дефолт «Команда N»)
- Настройки: время на ход (20–90 сек, шаг 5, дефолт 45), очков до победы (10–50, шаг 5, дефолт 20)
- Ход команды: Игрок 1 читает ослышку вслух ровным голосом (не подпевая!),
  Игрок 2 угадывает. После прочтения Игрок 1 больше не говорит — только жесты.
- Тап по карточке = переворот (ослышка ↔ ответ), можно туда-обратно
- Свайп карточки доступен ТОЛЬКО после первого переворота:
  вправо = верно (+1), влево = неверно (−1)
- Дублирующие кнопки ✗/✓ под карточкой (кликабельные) — для тех, кому жест неудобен
- Очки хода могут уходить в минус, но общий счёт команды не опускается ниже 0:
  `scores[i] = max(0, scores[i] + turnScore)`
- Карточки не повторяются, пока колода не кончится; потом перетасовка
- Победа: команда достигла целевого счёта ПОСЛЕ своего хода → экран победителя
- Ничья возможна (несколько команд с максимальным счётом)
- Пауза: кнопка в шапке, оверлей, таймер замирает
- Выход в меню: кнопка «←» на всех экранах кроме настройки, с подтверждением
  («Текущая игра и весь счёт будут потеряны»), таймер на время диалога замирает

## Экраны

1. **Setup** — заголовок, панель настроек (слайдеры команд/времени/очков,
   поля названий команд с цветной точкой и иконкой карандаша), кнопка «Начать игру»
2. **PreTurn** — счёт всех команд (активная выделена), «Ход команды X»,
   напоминание правил, кнопка «Начать ход»
3. **Turn** — плашка команды (цвет команды) + пауза; крупный таймер с
   прогресс-баром (последние 10 сек — красный + пульсация); очки хода;
   стопка карточек по центру; подсказки и легенда свайпа
4. **EndTurn** — счёт, «Время вышло», результат хода, кнопка далее
5. **Win** — победитель/ничья, финальный счёт, «Играть заново»

## Дизайн-система (из прототипа)

- Шрифт: **Inter** везде (400/500/600 текст, 700/800 заголовки и таймер),
  цифры табличные (`font-variant-numeric: tabular-nums`)
- Палитра: фон #F7F6F2, чернила #0D0D0D, карточка-вопрос ЧЁРНАЯ (#0D0D0D),
  карточка-ответ СИНЯЯ (#2F5FE0), верно #1FA97A, неверно #E24C4C
- Цвета команд: #E24C4C, #E0A72F, #1FA97A, #6B4CE0, #D6316C, #2F8FE0 (по кругу)
- Карточка: скругление 20px, мягкая тень, под ней серая «подложка» с поворотом
  3° (эффект стопки)
- Карточка-ответ: строка крупно, тонкая линия-разделитель, ниже исполнитель
  (жирнее) и название (тише)
- Микроанимации: spring cubic-bezier(.34,1.56,.64,1), 120–300мс, только
  transform/opacity; кнопки проседают при нажатии; поддержка prefers-reduced-motion
- Главная CTA-кнопка: градиент #2F5FE0 → #6B4CE0
- Фидбек свайпа: полупрозрачная радиальная виньетка по краям экрана
  (зелёная/красная), 300мс. Вибро НЕ используем (не работает на iOS в вебе)

## Известные грабли из прототипа (обязательно учесть)

1. **Прыжок карточки при перевороте.** Стороны разной высоты. Решение из
   прототипа: обе стороны рендерятся всегда, высота карточки = max(высот сторон),
   замеряется после рендера. В React это проще: рендерить обе стороны в
   grid-stack или замерять через ref + useLayoutEffect.
2. **Горизонтальный скролл при свайпе.** Контейнеру нужен overflow-x:hidden,
   а смещение карточки при вылете ограничивать (~260px), не 500+.
3. **Тап vs свайп на одном элементе.** Порог: движение >6px = drag, иначе тап.
   Свайп срабатывает при |dx| > 90px.
4. **Наложение плашки команды на кнопку «назад»** — шапке нужен отступ слева.
5. Таймер через setInterval: хранить id, чистить при паузе/выходе/размонтировании.

## Данные

Формат карточки (см. content/cards-titles.json):
```json
{ "mis": "как слышится", "artist": "...", "title": "...", "difficulty": "easy|medium|hard" }
```
Вариант с строчками добавляет поле "real". Контент будет пополняться владельцем
проекта вручную (в т.ч. через tools/misheard_helper.py — см. README там же).
Приложение должно читать карточки из отдельного JSON, не хардкодить.

## Стек (предложение, можно обсудить)

- React + Vite, одна страница, мобайл-фёрст (viewport ~390px)
- Состояние: обычный useState/useReducer, без внешних стейт-менеджеров
- Свайп: pointer events как в прототипе ИЛИ framer-motion (drag) — на выбор
- Деплой: статический хостинг (Vercel/Netlify/GitHub Pages)
- PWA-манифест сразу заложить (иконка, standalone) — задел на «установку» и
  будущий Capacitor-порт в сторы

## Что НЕ входит в первую версию

- Мультиплеер по сети, аккаунты, серверная часть
- Звук (обсуждали, отложено)
- Реальные тексты песен в контенте по умолчанию (правовой вопрос — на владельце)

## Роадмап после MVP

1. Экран выбора колоды (названия / строчки / своя колода)
2. Импорт своего JSON с карточками
3. Звуковой фидбек + haptics через Capacitor при портировании в сторы
